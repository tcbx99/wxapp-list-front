## 原先课程使用 Git 的要求
1. Git项目以组为单位创建，由组长将项目创建好。
2. 以小组为单位首先将项目结构搭建完成，提交到master分支，做好初始化
3. 完成master分支的初始化后，每个小组根据每次需求的提出以及小组成员分工的安排拉取新的分支，在各自的分支之上进行代码编写
（tips：每个分支对应的功能点尽量相对独立，不要多个分支对一个文件进行编辑，会出现冲突...）
4. 开发完成后将分支的内容合并到master
